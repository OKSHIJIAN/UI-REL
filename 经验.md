# UI-REL 插件问题排查经验

## 2026-05-12: 插件打开后显示原始代码文本（已解决）

### 现象
Figma 插件打开后，UI 界面不渲染，而是将 `ui.html` 的 JavaScript 源码作为纯文本显示在页面上。可以看到类似以下内容：
```
to prevent browser from closing the CURRENT script tag +' +
'/script><' + '/body><' + '/html>') var blob = new Blob([html], { ...
document.getElementById('frameSelector').className = 'frame-card';
...
```

### 根因（两层问题）

#### 问题 1：注释中的 `</script>` 导致浏览器提前关闭脚本块

`ui.html` 第 1867 行有一行注释：
```javascript
// Split </script> to prevent browser from closing the CURRENT script tag
+'<' + '/script><' + '/body><' + '/html>')
```

**浏览器的 HTML 解析器在解析 `<script>` 块时，不会做 JS 语法分析，只会做简单的字符串匹配。** 当解析器遇到 `</script>` 字符串（即使它在注释里或字符串字面量内），就会认为脚本块结束了。

这导致：
1. 第 1867 行注释中的 `</script>` 被当作闭合标签
2. 从第 1868 行到文件末尾的所有代码（包括真正的 `</script>`、`</body>`、`</html>`）被当作普通文本渲染
3. 用户看到的是截断后的 JS 源码

**注意：** 第 1940 行的真正 `</script>` 之前有前导空格（`    </script>`），这是第二个次要问题（见下方），但主要原因是第 1867 行的注释。

#### 问题 2：真正 `</script>` 标签有前导缩进

第 1940 行：
```html
    </script>   <!-- 有4个空格缩进 -->
```

虽然 HTML 规范允许空白字符，但某些环境下可能导致问题。

### 修复方案

**修复 1**（核心）：将注释中的 `</script>` 也拆分：
```javascript
// 修复前
// Split </script> to prevent browser from closing the CURRENT script tag

// 修复后  
// Split <' + '/script> to prevent browser from closing the CURRENT script tag
```

**修复 2**（辅助）：移除真正闭合标签的缩进：
```html
// 修复前
    </script>

// 修复后
</script>
```

### 技术要点

1. **HTML `<script>` 解析机制** — 浏览器用简单的字符串匹配找 `</script>`，不做 JS 语法分析。任何位置出现的 `</script>` 都会提前结束脚本块
2. **Figma 插件调试技巧** — 如果 UI 显示原始代码而非渲染界面，搜索所有 `</script>` 出现位置，检查是否有误匹配
3. **动态生成 HTML 的安全写法** — 在 JS 中需要输出包含 `</script>` 的 HTML 字符串时，必须拆分为字符串拼接：`'<' + '/script>'`
4. **不仅是代码中，注释中也一样** — 注释中的 `</script>` 同样会触发提前关闭

```bash
node --check extracted_code.js
```
